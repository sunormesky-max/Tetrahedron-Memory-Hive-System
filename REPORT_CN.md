## 一、设计理念与哲学

### 1.1 正四面体的几何意义

正四面体（Regular Tetrahedron）是柏拉图立体之一，在三维空间中具有独特的数学性质：

| 几何特性 | 数值 | 记忆系统对应 |
|----------|------|-------------|
| 顶点数 | 4 | 4个锚点 (A, B, C, D) |
| 面数 | 4 | 4种记忆类别 |
| 边数 | 6 | K4完全图，任意两点直达 |
| 对称性 | 高度对称 | 4面地位相等 |
| 稳定性 | 三角形是最稳定结构 | 记忆永不丢失 |

### 1.2 K4 完全图的意义

四面体的4个顶点构成 **K4完全图**（Complete Graph K4）：

```
        A
       /|\
      / | \
     /  |  \
    B---+---C
     \  |  /
      \ | /
       \|/
        D
```

**关键性质**：任意两个顶点之间都有直接连边，路径长度恒定为 1。

**记忆系统含义**：
- 任意两个记忆类别之间可以直接关联
- 检索复杂度 O(1)，不需要多跳
- 空间距离在几何上有明确意义

### 1.3 四面体四个面的记忆分类

```
FACE_ABC (技术/编程)     → 顶点 A-B-C 构成的面
FACE_ABD (生活/情感)     → 顶点 A-B-D 构成的面
FACE_ACD (工作/学习)     → 顶点 A-C-D 构成的面
FACE_BCD (其他/杂项)     → 顶点 B-C-D 构成的面
```

每个面是一个**等边三角形**，可以递归细分为更小的三角形，支持无限扩展。

### 1.4 核心设计原则

#### 1.4.1 记忆稳定性 vs 自然遗忘

```
┌─────────────────────────────────────────────────────┐
│                    传统方案                          │
├─────────────────────────────────────────────────────┤
│  • 模拟人类记忆的"自然遗忘"机制                      │
│  • 记忆重要性随时间衰减                              │
│  • 低重要性记忆被自动删除                            │
│  • 假设：遗忘是特性，不是缺陷                        │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│                   本系统设计                         │
├─────────────────────────────────────────────────────┤
│  • 正四面体结构 = 记忆稳定性                         │
│  • 记忆永不删除，只整合                              │
│  • 重要性用于排序，不用于淘汰                        │
│  • 假设：AI 不需要遗忘，需要整合                      │
└─────────────────────────────────────────────────────┘
```

**核心洞察**：
- 人类遗忘是**生物限制**，不是最优设计
- AI 的优势在于**永不遗忘** + **智能整合**
- 当记忆过多时，应该**合并相似记忆**，而非**删除记忆**

#### 1.4.2 O(1) vs O(log n) vs O(n)

| 复杂度 | 方案 | 适用场景 |
|--------|------|----------|
| O(1) | 哈希索引 | 精确匹配，本系统 |
| O(log n) | 向量数据库 | 语义搜索 |
| O(n) | 线性扫描 | 小数据集 |

本系统选择 O(1) 哈希索引作为主索引，语义搜索作为可选扩展。

## 六、已实现优化

### 6.1 查询缓存 (新增)

**文件**: core.py  
**位置**: `__init__()` 和 `recall()`

```python
# 新增属性
self._query_cache: Dict[str, Tuple[List[Dict], float]] = {}
self._cache_ttl: float = 60.0
self._cache_max_size: int = 1000

# recall() 中缓存检查
cache_key = f"{query_lower}:{category}:{limit}:{min_relevance}"
if cache_key in self._query_cache:
    cached_results, cached_time = self._query_cache[cache_key]
    if time.time() - cached_time < self._cache_ttl:
        return cached_results.copy()
```

**效果**: 214x 加速

### 6.2 无匹配查询优化 (修复)

**原代码** (有问题):
```python
if not candidate_ids:
    all_mems = sorted(self.memories.values(), key=lambda m: m.importance, reverse=True)
    candidate_ids = {m.id for m in all_mems[:self.MAX_RECALL_CANDIDATES]}
```

**修复后**:
```python
if not candidate_ids:
    logger.debug(f"No candidates found for query: {query}")
    return []
```

**效果**: O(n) → O(1)

### 6.3 面统计修复 (修复)

**原代码** (显示全0):
```python
for face_name, face in self.tetrahedron.faces.items():
    count = len(face.memory_ids)
```

**修复后**:
```python
from collections import Counter
face_counter = Counter(m.category.value for m in self.memories.values())
face_counts = {face_name: face_counter.get(f"FACE_{face_name}", 0) for face_name in ["ABC", "ABD", "ACD", "BCD"]}
```

| 项目 | 描述 |
|------|------|
| 现象 | total_memories=44 但 hub_memory_count=43 |

| 建议 | 添加 rebuild_index() 方法 |
