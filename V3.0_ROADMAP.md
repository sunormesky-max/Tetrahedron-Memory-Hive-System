# TetraMem-XL v3.0 Evolution Roadmap

**Base**: v2.3 (commit cd1f996) | **Date**: 2026-04-15

## Vision

Elevate TetraMem-XL from "pure geometric topology memory" to "intelligent self-organizing cognitive system". Core breakthroughs: LLM-powered dream synthesis + semantic-aware geometry + permanent ancestry + real-time visualization.

## Five Phases

### Phase 1: LLM Dream Synthesis (Week 1-3)

**Goal**: Replace template-based DreamProtocol with LLM-driven synthesis. Fusion quality 0.641 → 0.75+.

**Key deliverables**:
- `tetrahedron_memory/llm_integration.py` — LLMDreamExecutor with multi-provider support
- `tests/test_llm_dream.py` — unit + integration tests
- `demo_llm_dream_quality.py` — quality comparison demo

**Integration point**: `DreamProtocol(execute_fn=executor.execute, reflect_fn=executor.reflect)` in `tetra_dream.py` line 448-455. The DreamProtocol interface already accepts custom callables — no core changes needed.

**Providers supported**:
- Anthropic Claude (primary)
- OpenAI GPT (secondary)
- GLM-5 via OpenClaw (existing infrastructure)
- Local Ollama (offline fallback)

### Phase 2: Semantic-Aware Geometry (Week 4-6)

**Goal**: Replace SHA-256 hash mapping with embedding-based 3D projection. Semantically similar text gets spatially close coordinates.

**Key deliverables**:
- `tetrahedron_memory/semantic_geometry.py` — SemanticGeometryMapper
- `tests/test_semantic_similarity.py` — similarity preservation tests
- Comparison report: hash vs semantic mapping

**Integration point**: `text_to_geometry()` in `tetra_mesh.py` line 25-36. Currently a standalone function called by `start_api_v2.py`. Replace with pluggable mapper.

### Phase 3: Persistent Storage (Week 7-9)

**Goal**: Replace JSON file replay with SQLite + Parquet. 100% recovery after restart.

**Key deliverables**:
- `tetrahedron_memory/persistence_v2.py` — SQLite backend with Parquet export
- Schema design document
- Recovery verification script

**Integration point**: `startup()` and `_do_save()` in `start_api_v2.py`. Currently reads/writes `mesh_index.json`. Replace with SQLite backend.

### Phase 4: Ancestry Tracking (Week 10-11)

**Goal**: Track full ancestry chain through edge contractions. Complete Eternity guarantee.

**Key deliverables**:
- `tetrahedron_memory/ancestry.py` — AncestryTracker
- EternityAudit integration

**Integration point**: `edge_contraction()` in `tetra_mesh.py`. Record parent→child relationship when merging.

### Phase 5: Visualization (Week 12)

**Goal**: Real-time observability of mesh growth, dream synthesis, entropy evolution.

**Key deliverables**:
- `tetrahedron_memory/visualization.py` — OBJ export + Plotly charts
- FastAPI monitoring endpoints
- Dashboard demo
